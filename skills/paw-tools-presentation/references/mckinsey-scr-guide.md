# McKinsey SCR / Pyramid Principle Guide

## Core Principle

**Lead with the answer.** The Pyramid Principle reverses traditional narrative — instead of building up to a conclusion, you state your main recommendation first, then support it with reasoning.

## SCR Framework

### Situation

Establish shared context. What's true today that everyone agrees on?

- Current state of the business/market
- Relevant background facts
- What's working, what exists
- Sets the baseline

**Keep it factual and non-controversial.** This builds trust and establishes common ground.

### Complication

The trigger for action. What changed or what problem emerged?

- New challenge, opportunity, or gap
- Why the status quo is no longer acceptable
- The tension that demands response
- Creates urgency

**This is the "why now?"** Make it concrete and specific.

### Resolution

Your recommendation. What should be done?

- The proposed action or strategy
- Key supporting reasons (typically 3-5)
- Clear, actionable, and specific
- Answers the complication

**This is the "so what?"** Make it unambiguous.

## Pyramid Structure

```
                    ┌─────────────────┐
                    │  Governing Idea │
                    │   (Recommendation)  │
                    └────────┬────────┘
                             │
           ┌─────────────────┼─────────────────┐
           │                 │                 │
    ┌──────┴──────┐   ┌──────┴──────┐   ┌──────┴──────┐
    │ Key Argument│   │ Key Argument│   │ Key Argument│
    │     #1      │   │     #2      │   │     #3      │
    └──────┬──────┘   └──────┬──────┘   └──────┬──────┘
           │                 │                 │
    Supporting        Supporting        Supporting
      Data              Data              Data
```

## Application to Presentations

### Executive Summary Slide

The most important slide. It should answer:

1. **Where are we?** (Situation in one line)
2. **What's the issue?** (Complication in one line)
3. **What should we do?** (Resolution in one line)

### Supporting Slides

Each slide supports one key argument:

- **Situation slides:** Market data, current performance, baseline metrics
- **Complication slides:** Gap analysis, competitive threats, missed opportunities
- **Resolution slides:** Strategy options, recommended approach, implementation plan

## Common Patterns

### Strategy Recommendation

```
Situation: Market is growing at 15% annually
Complication: Our market share has declined 3% while competitors gained
Resolution: Invest in top-of-funnel awareness to recapture lost share
```

### Budget Request

```
Situation: Current campaign delivers 4.2% CVR
Complication: Awareness is underperforming, limiting conversion volume
Resolution: Increase paid media budget 40% for awareness campaigns
```

### Launch Decision

```
Situation: Product development is 90% complete
Complication: Competitor is launching a similar product in Q3
Resolution: Accelerate launch to Q2 to capture first-mover advantage
```

## Anti-Patterns

| Avoid | Instead |
|-------|---------|
| Building suspense | Lead with the answer |
| Chronological storytelling | Structure by argument |
| Multiple key messages per slide | One governing idea per slide |
| Hiding the recommendation | Put it on slide 1 |
| Too much situation | Situation should be brief, assumed knowledge |
